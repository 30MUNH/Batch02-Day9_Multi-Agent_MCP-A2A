2. Tool: Check Statute of Limitations

@tool
def check_statute_of_limitations(case_type: str) -> str:
    \"\"\"Kiểm tra thời hiệu khởi kiện dựa trên loại vụ án pháp lý.\"\"\"
    case_type = case_type.lower()
    if "civil" in case_type or "hợp đồng" in case_type:
        return "Thời hiệu khởi kiện tranh chấp hợp đồng dân sự là 03 năm."
    elif "labor" in case_type or "lao động" in case_type:
        return "Thời hiệu khởi kiện tranh chấp lao động là 01 năm."
    return "Thời hiệu khởi kiện phụ thuộc vào chi tiết cụ thể của vụ án."

Stage 5 Bonus Latency Optimization

Trong hệ thống A2A thực tế chạy 5 Agents, vấn đề lớn nhất là thời gian chờ (Timeout > 5 phút) do phải gọi chuỗi LLM quá nhiều lần.

❌ Trước khi tối ưu (LLM Routing)

Mất 5-15 giây chỉ để gọi OpenRouter xác định xem câu hỏi có thuộc vùng Thuế hay Pháp chế không.

# Tốn chi phí LLM và Latency
result = await llm.ainvoke(messages)
raw = result.content.strip()
parsed = json.loads(raw)
needs_tax = parsed.get("needs_tax")

✅ Sau khi tối ưu (Rule-based Routing)

Loại bỏ hoàn toàn bước gọi LLM, dùng keyword matching tốc độ ánh sáng (~0.001 giây).

# Tối ưu: Không dùng LLM, tốc độ O(n)
q = state["question"].lower()
needs_tax = any(kw in q for kw in ["tax", "thuế", "evasion"])
needs_compliance = any(kw in q for kw in ["sec", "aml", "fcpa"])

return {"needs_tax": needs_tax, ...}

Thành quả đo lường:

  • Hệ thống vượt qua lỗi Timeout của A2A Client.
  • Sử dụng hoàn toàn miễn phí qua openrouter/free.
  • Tổng thời gian phản hồi giảm xuống còn ~134 giây (cho một quy trình phức tạp gồm: Routing -> Law Analysis -> Parallel Tax/Compliance Analysis -> Final Aggregation).

Dòng chảy Dữ liệu (Timeline)

0.0s - User gửi câu hỏi đến Customer Agent.
0.1s - 3.0s - Customer Agent đánh giá intent, fetch Agent Card của Law Agent từ Registry và gửi request.
3.0s - 15.0s - Law Agent nhận request, dùng LLM phân tích "Contract Law" sau đó chạy Rule-based Routing phát hiện từ khóa "taxes", "regulatory". Kích hoạt LangGraph Send API để gọi song song các Sub-Agents.
15.0s - 75.0s (Xử lý song song) - Tax Agent và Compliance Agent cùng lúc nhận request. Hai agent phân tích các bộ luật độc lập trên OpenRouter.
75.0s - 78.0s - Law Agent tổng hợp (Aggregate) thông tin từ 2 Specialist Agents thành báo cáo thống nhất hoàn chỉnh.
78.15s - Trả kết quả cuối cùng cho người dùng thông qua terminal.
